RISC-V CPU Simulator 최적화 프로젝트 (RISCV-Sim Optimizer)
1. 개요

    프로젝트 목적: 컴퓨터구조개론 과정에서 구현한 RISC-V CPU Simulator 바이너리의 성능 병목 지점을 프로파일링 도구(gprof)로 진단하고, 소프트웨어 및 컴파일러 레벨 최적화를 적용하여 실행 시간을 단축함.

    타겟 프로그램 기능: RISC-V Instruction 바이너리를 입력받아 x0~x31 레지스터의 최종 상태를 연산하고 출력하는 시뮬레이터(add, sub, lw, sw 등 총 26개 명령어 지원).

2. 병목 지점 분석 (Profiling)

    초기 컴파일 규격: g++ -W -pg -Og -o riscv-sim main.cc 환경에서 런타임 분석 수행.

    오버헤드 요인: gprof 분석 결과 std::_Base_bitset 관련 내부 함수 및 to_string() 함수의 호출 빈도가 가장 높았으며, 호출 횟수 누적으로 인한 과도한 self-seconds 소모가 확인됨.

    원인 진단: 루프(Loop) 내부의 명령어 분석 및 디코딩 로직에서 std::bitset과 std::string 간의 잦은 형변환 및 비교 연산이 치명적인 병목을 유발함.

3. 단계별 최적화 프로세스 (Optimization Steps)
1단계: 데이터 구조 개선 (String/Bitset -> Integer)

    수행 내용: 루프 내부에서 구문 분석에 활용되던 오버헤드가 큰 std::string 및 std::bitset 연산을 모조리 제거하고, unsigned int 기반의 비트 연산(Bitwise Operations)으로 대체함.

    결과: 초기 변환식 및 immediate 생성 로직의 정수 연산 최적화를 통해 메인 함수 총 실행 시간이 기존 대비 약 55배 감소하는 가장 큰 성능 개선을 달성함.

2단계: 인라인 함수 도입 (Inline Functions)

    수행 내용: 무한 루프 내에서 명령어 세트를 지속적으로 추출하는 오버헤드를 제어하기 위해 getinst를 포함한 핵심 하위 함수들에 inline 키워드를 적용함.

    결과: 코드 변경 후에도 기존 -Og 컴파일러 플래그 하에서는 성능 개선이 전혀 나타나지 않음. 분석 결과 디버깅 옵션(-Og) 특성상 컴파일러가 유저의 inline 요청을 무시하고 확장하지 않는 것으로 파악됨.

3단계: 컴파일러 최적화 플래그 제어 (-Og -> -O1)

    수행 내용: 컴파일러가 인라인 함수 확장을 정상적으로 처리할 수 있도록 빌드 플래그를 최적화 1단계 옵션인 -O1으로 승격하여 다시 빌드를 전개함 (g++ -W -pg -O1 -o riscv-sim main.cc).

    결과: 컴파일러 최적화와 인라인 함수가 정상 결합되어 함수 호출 오버헤드가 완벽히 제거됨에 따라, 전체 시뮬레이터 실행 시간이 0초에 가깝게 단축되는 최종 최적화를 달성함.

4. 최종 결론

    자료구조 최적화의 중요성: 고성능이 요구되는 시뮬레이터 루프 내부에서는 무거운 객체(string, bitset)의 동적 할당 및 변환을 피하고 하드웨어 친화적인 기본 정수형(int) 연산을 사용하는 것이 성능에 직결됨.

    컴파일러 옵션 상호작용: 소스 코드 레벨의 인라인화 기법은 컴파일러의 최적화 레벨 옵션(-O1 이상)과 연계되어야만 실질적인 오버헤드 감소 효과를 낼 수 있음을 실험적으로 증명함.
